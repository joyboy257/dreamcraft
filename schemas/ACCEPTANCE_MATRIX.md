# Acceptance Matrix

| Product promise | Implementation evidence | Automated evidence | Manual evidence |
|---|---|---|---|
| Dream becomes a world | Live generation provider + DreamSpec compiler | Schema/integration test | Novel prompt video |
| World is recognizable | Blueprint anchors + staging compiler | Three anchor references and in-bounds placements | First-view screenshot/rubric |
| World is playable | Shell, spawn repair, player motor | Safe-spawn and browser completion tests | Full 60–90s run |
| Entities are recognizable | EntityKit readability contract | Readability score assertions | Hero screenshots at distance |
| Physics feel dream-specific | Physics DSL and transitions | Clamp/resolution tests | Contrasting dream clips |
| Story is open-ended | DreamPlayGraph | Reachable ending and reference tests | Lottery/social dream playthrough |
| Generation is safe | No eval, strict validation | Malicious/over-budget fixtures | Security review |
| Demo survives API failure | Local fragment and cached samples | API-disabled E2E | Offline demo clip |
| Performance is credible | Chunk meshing, workers, quality tiers | Metrics thresholds where stable | Profiler capture on demo device |
| Codex use is substantive | Sol root + worker configs + build log | Repository history/docs | Subagent/build screenshots |
| GPT-5.6 use is substantive | Runtime dream compiler | Provider tests and metadata | Live prompt generation |
| Judges can run it | README/deployment | Fresh-clone CI/build | Incognito/second-device smoke |
